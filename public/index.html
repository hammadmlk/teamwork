Teamwork.js

framework for distributed computing on web browsers

What does that mean?

What is Teamwork.JS

Any web browser displaying a web page with teamwork.js
becomes part of a bot-net. Each node in this bot-net has two-way
communication with the master and will run any JS code distributed
by the master.

Teamwork.js is a framework for distributed computation using the
resources of website visitors

.
What motivates us to make this?

What Motivates Us

The past decade has seen a major shift from vertical scaling to horizontal scaling. Most modern day computational problems are parallel in nature and usually solved via coordination between distributed systems. For most of these solutions, the only communication requirement is two way communication between the server and the clients, and/or peer to peer communication between clients.

Modern web browsers are capable of handling extensive computations (benchmarks) as well as perform real two-way communication (WebSockets) and peer-to-peer communication (WebRTC). This makes them capable of being used in such distributed settings to solve hard problems.

As alternatives to native programs, web-browser based solutions have lower entry barriers. Volunteers can join the system by just opening a URL; or be forced to join by appending the JavaScript code to the html page served (think ad-networks and proxy services).

This means access to millions of compute nodes. Precisely what motivates us.

This project will develop a framework for such web-browser based distributed computation and evaluate the effectiveness of using it as an alternate to native code based systems like the very famous SETI-at-home project.

Motivation 1

The every superior web browsers

Computational power of JavaScript: Recent competition between leading web browser developers has resulted in extremely efficient JavaScript engines.Benchmarks suggest that JavaScript computational performance is not far behind Java.

Real support for two way communication and peer to peer:WebSockets have allowed efficient, low-latency, real two way communication between browser and server. And now with WebRTC peer-to-peer communication between two browsers is possible.

Ability to run you code on millions on machines: Public proxies and ad-networks serve custom pages to millions of users and have the ability to append any piece of JavaScript to the pages they serve.
