Teamwork.js

framework for distributed computing on web browsers

What does that mean?

What is Teamwork.JS

Any web browser displaying a web page with teamwork.js
becomes part of a bot-net. Each node in this bot-net has two-way
communication with the master and will run any JS code distributed
by the master.

Teamwork.js is a framework for distributed computation using the
resources of website visitors

.
What motivates us to make this?

What Motivates Us

Possibility of having millions of free compute nodes

Background

The past decade has seen a major shift from vertical scaling to horizontal scaling. Most modern day computational problems are parallel in nature and usually solved via coordination between distributed systems. For most of these solutions, the only communication requirement is two way communication between the server and the clients, and/or peer to peer communication between clients.

Modern web browsers are capable of handling extensive computations (benchmarks) as well as perform real two-way communication (WebSockets) and peer-to-peer communication (WebRTC). This makes them capable of being used in such distributed settings to solve parallel problems.

Benefits

As alternatives to native programs, web-browser based solutions have lower entry barriers. Volunteers can join the system by just opening a URL; or be forced to join by appending the JavaScript code to the html pages served (think ad-networks and proxy services).

This means the feasibility of having millions of free compute nodes. Think of the possibilities: precisely what motivates us.

What is ahead

This project will develop into a framework for web-browser based distributed computation. Programming models like MapReduce will be supported; developers will be able to submit custom compute-functions that the system will automatically distribute and aggregate.

Current state of teamwork.js?

Current State

To demonstrate the validity of the idea, at the moment we are using teamwork.js to crack passwords (embarrassingly parallel).

Performance

We measured the performance of the system with variable number of nodes. The results are promising, for each new machine added the performance increases by a fixed constant.

* when using machines with identical hardware and software.
